변경된 코드(또는 지정된 파일)에 대해 구조화된 코드 리뷰를 수행하라.

## 리뷰 체크리스트

### 1. CRITICAL (반드시 확인)
- [ ] API Key/토큰/비밀번호가 하드코딩되어 있지 않은가?
- [ ] `!` 연산자가 안전하게 사용되었는가?
- [ ] try-catch가 적절히 사용되었는가?
- [ ] `dynamic` 타입이 남용되지 않았는가?

### 2. HIGH (중요)
- [ ] 하나의 위젯이 200줄을 넘지 않는가?
- [ ] 비즈니스 로직이 UI에서 분리되어 있는가?
- [ ] `build()` 안에서 무거운 연산이 없는가?
- [ ] 긴 리스트에 `ListView.builder`를 사용했는가?

### 3. NORMAL (권장)
- [ ] 파일은 `snake_case`, 클래스는 `PascalCase`인가?
- [ ] 변하지 않는 위젯에 `const` 키워드를 사용했는가?
- [ ] 불필요한 주석이나 `print()` 문이 있는가?
- [ ] 반복되는 코드가 있는가?

### 4. UI/UX (약비서 전용)
- [ ] UI 텍스트가 한국어로 되어 있는가?
- [ ] 4050 타겟에 맞는 텍스트 크기인가?
- [ ] 로딩/에러/빈 상태가 처리되어 있는가?

## 결과 출력 형식

```markdown
## 코드 리뷰 결과

### CRITICAL (수정 필수)
- [파일명:라인] 이슈 설명 + 수정 제안

### HIGH (권장 수정)
- [파일명:라인] 이슈 설명

### NORMAL (선택 개선)
- [파일명:라인] 이슈 설명

### 잘된 점
- 칭찬할 부분

### 요약
- Critical: N개 / High: N개 / Normal: N개
- 전체 평가: [통과 / 조건부 통과 / 재검토 필요]
```

**판정 기준:**
- Critical 1개 이상 → 재검토 필요
- High 3개 이상 → 조건부 통과
- 그 외 → 통과
